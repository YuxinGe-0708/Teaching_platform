## 1 范围

### 1.1 标识

**系统名称**：在线教学与实训平台

**文档名称**：在线教学与实训平台软件需求规格说明书

**项目类型**：高校教学与实训一体化平台

**目标用户**：教师、学生、系统管理员

### 1.2 系统概述

本系统面向高校教学与实训场景，提供课程管理、在线学习、在线实验、自动评测、作业布置与批改、成绩统计、公告通知等功能，帮助教师更高效地组织教学，帮助学生更便捷地完成学习与实训任务。

系统采用面向对象的设计思想进行功能拆分，按照教师端、学生端、管理员端、评测服务端进行模块化组织，保证后续开发、测试与维护的可扩展性。

### 1.3 文档概述

本文档用于描述系统的业务目标、功能需求、接口需求、数据需求、非功能需求以及验收标准等内容，作为后续概要设计、详细设计、开发实现、测试与交付的依据。

### 1.4 基线

本需求规格说明书的基线包括：

1. 当前已确认的业务范围：在线教学、在线实训、评测、成绩统计与管理。
2. 当前已确认的技术路线：Java 8、Spring Boot、Maven、MySQL。
3. 当前已确认的角色体系：教师、学生、系统管理员。
4. 当前已确认的开发计划：分阶段迭代开发、测试、部署与交付。

## 3 需求

<a id="sec-3-1"></a>
### 3.1 所需的状态和方式

系统应支持以下工作状态：

- **未登录状态**：访问平台首页，可查看平台介绍、教学空间、资源广场、帮助中心等入口。
- **已登录学生状态**：进入学生首页，查看个人课程列表、课程选择与退课、班级管理、在线作业、在线考试等功能。
- **已登录教师状态**：进入教师首页，查看已创建课程列表、作业布置与批改、成绩统计等功能。
- **管理员状态**：进入管理后台，进行账号管理、公告管理与系统监控。

系统的工作方式应支持浏览器访问、页面跳转、表单提交、异步请求、登录身份识别与权限控制。

<a id="sec-3-2"></a>
### 3.2 需求概述

#### 3.2.1 目标

开发意图与应用目标：本系统旨在提供一个综合的在线教学与实训平台，解决现有教学平台在课程管理、在线实验、自动评测、学习过程管理和师生互动方面的不足。系统将整合教学空间、资源广场、帮助中心、课程学习、在线作业、在线考试和成绩统计等功能，提供一体化服务，提高教学效率与学习体验。同时引入AI功能模块，帮助学生完成知识点的讲解与梳理。

本项目的具体目标包括：

1. 建立统一的平台首页入口，满足未登录用户浏览平台概览、登录与进入教学空间的需求。
2. 为学生提供清晰的学习入口，支持查看个人课程、选退课、在线作业、在线考试、成绩查看与通知查看。
3. 为教师提供完整的教学管理入口，支持创建课程、管理课程成员、发布作业与考试、批改作业、查看成绩统计。
4. 为系统管理员提供基础管理入口，支持用户管理、公告管理、系统配置和运行状态查看。
5. 为在线实训能力提供可扩展的评测支撑，支持后续接入代码评测服务与智能辅助能力。
6. 为AI相关功能提供支撑，支持AI整理课程知识点，一键导出AI笔记等功能。

主要功能和流程：

- 平台首页展示教学空间、资源广场、帮助中心，并提供右上角登录入口。
- 用户根据身份登录后自动跳转到学生首页或教师首页，首页菜单根据身份动态展示。
- 学生可查看我的课程、课程选择与退课、班级管理、在线作业、在线考试、成绩统计等栏目。
- 教师可查看已创建的课程管理、作业布置与批改、成绩统计、班级管理等栏目。
- 管理员可进行用户、公告与系统配置管理，并查看系统运行日志。
- 平台支持课程列表展示、作业提交、成绩查询、通知展示与基础管理功能。

系统高层次图与关系：
系统由用户界面层、业务服务层、数据管理层与评测服务层组成，可与后续的更多教学工具或智能辅助模块扩展集成。系统设计强调“角色驱动 + 模块化拆分”，保证前后端联调时边界清晰、接口可追踪。

#### 3.2.2 运行环境

- **硬件环境**：服务器应支持中等并发访问，满足教学场景下的稳定运行需求。若后续接入外部评测 API 或大模型 API，服务器应具备稳定的出网能力和基本的异常重试能力。
- **客户端环境**：支持常见 Windows、macOS、Linux 设备上的浏览器访问；学生和教师端页面应在常见分辨率下保持可用。
- **支持环境**：需要数据库系统支持课程、用户、作业、成绩、日志等信息的存储与快速查询；后端可部署于本地服务器或云服务器。
- **浏览器兼容性**：支持主流浏览器的正常访问与页面渲染，页面表单、菜单导航和登录跳转不应依赖单一浏览器特性。
- **开发环境**：前后端开发应在统一的代码仓库和统一的配置规范下进行，便于后续联调、测试和部署。

#### 3.2.3 用户的特点

本系统面向高校教学场景中的师生及管理人员。

- **学生**：注重课程学习、作业提交、考试与成绩查看，希望界面清晰、流程简单、响应快速。学生通常在移动端和浏览器中使用系统，关注“少点击、少跳转、结果明确”。
- **教师**：关注课程创建、资源管理、作业布置、批改与成绩统计，希望功能集中、操作高效。教师通常以课程为中心组织教学活动，关注“课程管理闭环是否完整”。
- **管理员**：关注账号管理、公告发布、系统运行状态与基础配置，倾向于快速查看和批量处理。
- **开发人员**：需要结构清晰、接口规范、模块可复用，便于多人并行开发。

#### 3.2.4 关键点

**关键功能**：

- 平台首页、登录与身份识别
- 学生首页与教师首页差异化展示
- 课程管理与课程列表展示
- 课程选择与退课
- 班级管理
- 在线作业
- 在线考试
- 作业布置与批改
- 成绩统计
- 通知与公告
- 用户身份和权限控制
- 评测服务接入与结果回传
- AI 辅助学习能力（后续可扩展）

**关键技术和算法**：

- Web 开发技术：前端采用 Vue 或模板渲染方式实现页面展示，后端采用 Spring Boot 实现业务逻辑。
- 数据库技术：采用 MySQL 存储用户信息、课程信息、作业信息、成绩信息和日志信息。
- 身份识别算法：根据登录账号角色判断是学生还是教师，并自动跳转到对应首页。
- 课程列表查询算法：根据用户身份从课程表中查询已选课程或已创建课程。
- 作业状态管理：根据提交状态、截止时间和批改状态进行展示。
- 消息通知技术：使用站内消息或页面通知形式提示登录、作业、考试与成绩变化。
- 评测结果处理：若后续接入 Judge API / 代码评测服务，应支持提交代码、回传结果、状态轮询和错误提示。
- AI 辅助能力：若后续接入大模型能力，应支持知识点解释、内容摘要、结构化整理等功能。

#### 3.2.5 约束条件

- **开发期限**：项目计划在两个月左右完成。
- **经费约束**：先不引入复杂商业组件和数据库服务，优先实现核心功能。
- **技术约束**：采用成熟、轻量、易维护的技术栈；前后端接口应统一定义，避免后续多人开发时出现接口冲突。
- **安全约束**：遵守基本的数据保护要求，避免泄露用户信息。
- **协作约束**：数据库、接口返回格式、命名规范与错误码必须统一，便于多人协作。

<a id="sec-3-3"></a>
### 3.3 需求规格

#### 3.3.1 软件系统总体功能/对象结构

系统总体对象可划分为：

- **用户对象**：学生、教师、管理员
- **课程对象**：课程信息、课程成员、课程内容
- **作业对象**：作业任务、提交记录、批改结果
- **考试对象**：考试安排、试卷、答题记录、成绩
- **通知对象**：站内通知、公告、消息记录
- **权限对象**：角色、菜单、页面访问权限
- **评测对象**：提交任务、评测结果、状态记录、错误日志
- **AI 对话对象**：提问内容、回答内容、会话记录、使用日志

#### 3.3.2 软件子系统功能/对象结构

**学生子系统**：

- **核心功能模块**
  - **选课与课程模块**：负责按列表或编号检索课程、加入与退选课程，展示已选课程清单、课程进度、课程简介、教师信息及退选课日志。
  - **班级信息模块**：负责展示学生所属班级的基础信息、班级成员列表以及班级公告通知。
  - **在线作业模块**：负责展示课程作业列表与详情，支持学生提交作业内容并查看教师的批改结果。
  - **在线考试模块**：负责展示考试安排与纪律要求，提供在线答题环境及考试结果的提交。
  - **成绩查询模块**：负责分类展示学生的课程成绩、作业成绩、考试成绩及相关的数据统计信息。
  - **消息通知模块**：负责接收与展示课程更新、作业发布、批改结果及系统级通知。
  - **日常学习辅助模块**：负责提供课程直录播与PPT查看，支持随堂记录电子笔记、调用AI进行实时答疑，以及一键导出笔记生成思维导图和生成课程总结。
  - **答疑与交互模块**：负责支持学生发布匿名提问并定向推送给助教/教师，支持在班级/课程交流平台分享见解、学习心得与共享优秀学习资源。

- **学生端核心对象**

  - **CourseSelection (选课记录对象)**：选课关系凭证，属性包括：学生学号、课程编号、选课状态（在学/已退选）、选/退课时间、当前课程进度。
  - **StudentAssignment (作业视图与提交对象)**：作业执行体，属性包括：作业编号、作业详情要求、提交内容（文本/文件）、提交时间、批改状态、批改结果/评语。
  - **ExamRecord (考试记录对象)**：考试答题实体，属性包括：考试编号、考试要求说明、开始答题时间、交卷时间、答题内容、考试状态（未考/答题中/已交卷）。
  - **ScoreTranscript (学生成绩单对象)**：成绩数据视图，属性包括：学生学号、关联课程、平时作业成绩、考试成绩、最终总成绩、成绩排名与统计信息。
  - **Notification (消息通知对象)**：消息触达载体，属性包括：通知类型（课程更新/作业/系统提醒等）、标题、内容、发送时间、已读/未读状态。
  - **StudyNote (学习笔记对象)**：个人学习辅助资产，属性包括：笔记ID、关联课程/课件、笔记内容、记录时间、AI总结快照、导出的思维导图文件路径。
  - **InteractionPost (互动问答帖对象)**：社区交流载体，属性包括：帖子ID、发帖人（支持匿名标记）、帖子类型（向老师提问/心得分享/资源共享）、帖子内容/附件、回复列表、发布时间。

**教师子系统**：
- **核心功能模块**
  - **课程中心模块**：负责课程的生命周期管理。
  - **班级人员模块**：负责管理选课学生及班级组织。
  - **教学资源模块**：负责课件、视频、参考资料的管理。
  - **作业实训模块**：负责布置任务、设置评测规则及监控进度。
  - **批改评分模块**：负责对学生提交的内容进行评判与反馈。
  - **数据统计模块**：负责成绩汇总、分析及报表生成。

- **教师端核心对象**

  - **Course (课程对象)**：核心容器，属性包括：课程名、课程编号、学时、学分、课程封面图、状态（草稿/开启/结课）。
  - **Class (班级/小组对象)**：逻辑分组，属性包括：班级名称、人数上限、加入邀请码。
  - **TeachingResource (教学资源对象)**：文件实体，属性包括：文件名、文件类型（PPT/PDF/Video）、文件大小、下载次数、所属章节。
  - **Assignment (作业/任务对象)**：任务定义，属性包括：标题、描述、开始时间、截止时间、满分值、任务类型（普通作业/编程实训）。
  - **TestCase (测试用例对象)**：针对编程实训，属性包括：输入数据、预期输出数据、分值权重。
  - **StudentSubmission (学生提交对象)**：成果记录，属性包括：提交代码/文档、提交时间、运行状态（AC/WA/PE等）、系统初步打分。
  - **GradeReport (成绩统计对象)**：结果视图，属性包括：学生学号、各阶段得分、总排名。

**管理员子系统**：

- **核心功能模块**：
   - **用户管理模块**：负责师生账号的全生命周期管理，包括批量导入、信息维护、密码重置与禁用/启用。
   - **角色与权限模块**：负责角色分配与基础权限控制，确保不同角色（学生/教师/管理员）拥有对应的操作范围。
   - **公告管理模块**：负责平台级公告的发布、编辑、置顶与删除。
   - **系统配置模块**：负责维护系统基础参数、首页布局、菜单显示等全局配置。
   - **日志与监控模块**：负责系统运行日志的查询与简要监控（在线用户数、评测队列长度等）。

- **管理员端核心对象**：
   - **用户账户对象**
	属性：用户ID、用户名、密码（加密存储）、真实姓名、角色（学生/教师/管理员）、邮箱、手机号、状态（正常/禁用）、注册时间、最后登录时间。
   - **角色对象**
	属性：角色ID、角色名称（如学生、教师、管理员）、角色描述、权限标识列表（如 `course:create`, `user:manage` 等，可采用简单字符串集合）。
   - **公告对象**
	属性：公告ID、标题、内容、发布者、发布时间、是否置顶、状态（生效/失效）、目标范围。
   - **日志对象**
	属性：日志ID、日志类型（登录/操作/异常）、操作用户、操作描述、操作时间、IP地址、请求路径（可选），结果状态（成功/失败）。
   - **系统配置对象**
	属性：配置项键（如 `allow_registration`, `homepage_layout`）、配置项值、配置描述、最后修改者、修改时间。

**评测/AI 支撑子系统**：

- 评测服务：接收提交内容、执行评测并回传结果。
- AI 辅助服务：接收知识提问、生成解释内容与结构化摘要。
- 任务记录：保存调用日志、失败原因与重试记录。

#### 3.3.3 描述约定

- 使用统一的 UML 符号描述系统架构和流程。
- 接口请求统一封装，避免在多个页面重复编写相同请求逻辑。
- 重要变量、状态字段和枚举值应建立统一说明文档。
- 页面命名、菜单命名与接口命名保持一致，便于维护和定位。
- 后端数据实体、接口返回结构应统一定义，禁止各模块自行随意命名。
- 对于外部评测或 AI 能力，调用参数、返回值和异常处理方式必须统一约定。

<a id="sec-3-4"></a>
### 3.4 CSCI能力需求

#### 3.4.1 学生课程管理能力
- 在主页中，能够通过快捷菜单直接跳转到“我的课程”、“选课服务”、“班级管理”、“作业库”、“交流平台”等五个基础功能模块，并能够查看最近通知（小红点提示）
- 能够查看“我的课程”列表，并支持按课程名称、学期、教师进行基础筛选
- 在课程主页，能够查询课程详情、教师信息、课程简介、课程直录播、课程通知、课程资源与课程作业。能够查看课程下的班级与学习进度信息，并支持展示课程当前学习任务。在课程直录播界面，能够记录随堂笔记，使用实时AI答疑，生成课程总结，导出思维导图
- 在“选课服务”界面，能够进行课程选择与退课操作，系统需提示操作结果与当前课程状态。
- 在“班级管理”界面，能够加入班级，查看班级基础信息、班级成员、班级通知
- 在“作业库”界面，能够查看当前作业，作业详情，提交作业，查看作业批改结果（客观题自动判分，主观题拍照上传，编程题集成轻量级WebIDE，返回ACWA/CE/RE等判题结果和解析），使用AI实时作业答疑
- 在“交流平台”界面，能够在班级/课程互动区自由发表学习心得、疑问，并与其他同学、老师、助教开展线上讨论

#### 3.4.2 平台首页与登录能力

- 平台首页应展示教学空间、资源广场、帮助中心等入口，并提供明显的登录按钮。
- 用户登录后系统应根据身份跳转到学生首页或教师首页。
- 若登录身份未识别或权限不匹配，应给出明确提示并阻止进入错误页面。

#### 3.4.3 教师课程管理能力
本节详细描述教师在平台中的各项操作能力，涵盖所有 CRUD 需求。

- 增加课程：教师可录入课程名称、课程描述、选择所属学科分类，上传课程封面，并设置该课程是否允许学生自主申请加入。
 - 修改课程：教师可更新课程简介、更换封面图片，或切换课程状态（如从“进行中”改为“已结课”）。
 - 查询课程：支持按课程名称关键词搜索，或按课程创建时间、学生人数进行排序展示。
  - 删除课程：教师可删除未激活的课程；对于已有学生选课的课程，支持“归档”处理（逻辑删除），保留历史数据但不显示在活跃列表中。
  - 增加班级：教师可在某门课程下创建多个平行班（如：一班、二班），并生成专属邀请码。
  - 修改班级：修改班级名称或人数上限。
  - 查询学生：教师可按班级查看学生名单，查询特定学生的学号、姓名、加入时间及在线学习时长。
  - 删除/移除学生：教师可将误操作进入班级的学生移出课程。
  - 增加资源：教师可上传 PPT、PDF 课件或配套代码包，支持批量上传。
  - 修改资源：支持修改资源文件名称、调整所属章节、重传新版本覆盖旧版本。
  - 查询资源：支持按资源类型（视频/文档）过滤，查看资源的被下载次数。
  - 删除资源：教师可单选或批量删除过期的教学资料。
  - 增加任务：
	- 基础任务：设置标题、作业内容说明、附件下载、截止时间。
	- 实训任务：需额外录入代码框架（模板）以及多个测试用例。
  - 修改任务：在任务未截止前，教师可修改题目说明或延长截止日期。
  - 查询任务：教师可查看该课程下所有已发布、待发布、已过期的任务列表。
  - 删除任务：教师可撤回或删除错误发布的作业任务。
  - 增加评分：教师对学生提交的主观题给出分数和文字评语；对于编程题，教师可手动覆盖自动评测的分数。
  - 查询提交记录：教师可按任务查看全班提交详情（已交/未交/待批改），支持在线查阅学生提交的代码或文档。
  - 修改分数：支持对已批改的分数进行申诉重改。
  - 成绩查询与统计：
	- 支持查询单名学生的所有成绩单。
	- 支持查询班级平均分、最高分、最低分及分数段分布直方图。
  - 删除/清空数据：教师支持清空测试账号产生的提交记录。
  - 增加通知：教师发布课程重要通知（如：调课、作业延期提醒），支持推送到学生首页。
  - 修改/查询通知：编辑已发通知的内容，或查看历史通知记录。
  - 删除通知：教师可删除陈旧的系统公告。

#### 3.4.4 作业与考试能力

- 教师能够布置作业和考试，设置标题、说明、截止时间、提交要求和发布范围。
- 学生能够查看、提交、补交作业以及参与考试，系统应记录提交状态和提交时间
- 学生应在规定时间内完成并提交作业或考试，若超时，系统应自动提交已作答内容。
- 系统能够记录提交时间、批改状态、评分结果、评语和历史提交记录。
- 系统应支持基础的成绩反馈，并能按课程或作业查看结果。
- 系统应支持编程题自动评测，通过对接外部代码评测服务，返回 AC/WA/CE/RE/TLE 等判题结果。

#### 3.4.5 消息通知能力

- 系统公告、课程更新、作业发布、作业批改、考试安排等重要事件应有通知提示。
- 通知采用站内消息形式，内容包括标题、简要正文和时间戳。
- 支持在首页或个人中心中查看最近通知。
- 通知应支持“未读/已读”状态，并按重要程度分类，便于用户追踪重要信息。

#### 3.4.6 评测与智能辅助能力

- 系统后续应支持接入代码评测服务，对编程作业或实验进行自动判定。
- 系统后续应支持接入大模型能力，提供知识点解释、内容总结、结构化输出等辅助功能。
- 若相关能力暂未接入，应在界面和接口层预留扩展位置。

<a id="sec-3-5"></a>
### 3.5 CSCI外部接口需求
#### 3.5.1 前端UI交互接口
本类接口为前端页面渲染、页面跳转、权限判断提供支撑，仅负责页面数据初始化与身份适配，不处理核心业务逻辑。
1. **UI-001 平台首页接口**
- 请求方式：GET
- 接口功能：渲染平台未登录首页，返回平台公告、功能入口、基础配置信息
- 权限要求：无需登录，全开放访问
- 响应数据：首页轮播公告、教学空间入口、资源广场入口、帮助中心入口、全局基础配置
2. **UI-002 登录接口**
- 请求方式：POST
- 接口功能：完成账号密码校验、身份识别，生成登录token，返回用户角色与跳转目标页面
- 权限要求：无需登录，支持账号密码登录
- 请求参数：用户名、密码、验证码
- 响应规则：校验通过返回token、用户角色、跳转地址；校验失败返回错误提示，连续5次错误锁定账号10分钟
3. **UI-003 学生首页接口**
- 请求方式：GET
- 接口功能：渲染学生登录后首页，返回个人信息、待办事项、课程列表、未读通知
- 权限要求：仅已登录学生角色可访问
- 响应数据：学生个人信息、最近学习课程、待提交作业、未读通知、快捷功能入口
4. **UI-004 教师首页接口**
- 请求方式：GET
- 接口功能：渲染教师登录后首页，返回授课课程、待批改作业、成绩统计概览、未读消息
- 权限要求：仅已登录教师角色可访问
- 响应数据：教师个人信息、授课课程列表、待批改作业统计、班级学情概览、快捷操作入口
- `UI-001`：平台首页接口
- `UI-002`：登录接口
- `UI-003`：学生首页接口
- `UI-004`：教师首页接口
- `API-001`：课程列表查询接口
- `API-002`：课程选择/退课接口
- `API-003`：作业查询与提交接口
- `API-004`：作业布置与批改接口
- `API-005`：成绩统计接口
- `API-006`：通知查询接口
- `API-007`：用户与角色查询接口
- `API-008`：评测结果查询接口
- `API-009`：AI 辅助问答接口
#### 3.5.3 核心服务HTTP接口
本类接口为系统核心业务功能提供支撑，完全匹配3.4节能力需求，每个接口明确业务规则、权限约束与数据范围。
1. **API-001 课程列表查询接口**
- 请求方式：GET
- 接口功能：按用户角色返回对应课程列表，学生返回已选课程，教师返回已创建课程，支持筛选、分页、排序
- 权限要求：已登录用户，仅可查看自身权限范围内的课程数据
- 请求参数：分页参数、筛选条件（课程名称、学期、状态）
- 业务规则：学生仅可见自己选课的课程，教师仅可见自己创建的课程，管理员可见全平台课程
2. **API-002 课程选择/退课接口**
- 请求方式：POST
- 接口功能：完成学生选课、退课操作，校验课程状态、选课容量、重复选课规则
- 权限要求：仅已登录学生角色可访问
- 请求参数：课程ID、操作类型（选课/退课）
- 业务规则：已结课课程禁止选课，退课后保留历史选课记录，禁止重复选课，操作完成同步更新选课记录与通知
3. **API-003 作业查询与提交接口**
- 请求方式：GET/POST
- 接口功能：查询学生作业列表、作业详情，提交作业内容、附件、编程代码，记录提交时间与状态
- 权限要求：仅已登录学生，仅可操作自己所选课程的作业
- 业务规则：截止时间后禁止提交，系统自动保存草稿，超时自动提交已作答内容，保留所有历史提交记录
4. **API-004 作业布置与批改接口**
- 请求方式：GET/POST
- 接口功能：教师发布/修改/删除作业，查询学生提交记录，在线批改作业、打分、填写评语，手动修改自动评测分数
- 权限要求：仅授课教师，仅可操作自己课程下的作业
- 业务规则：批改完成同步推送通知给学生，支持批量批改、分数复核，已发布作业未截止可修改
5. **API-005 成绩统计接口**
- 请求方式：GET
- 接口功能：按课程、作业、班级维度查询成绩详情、分数统计、排名、分段分布
- 权限要求：学生仅可查看自己的成绩，教师可查看所带班级/课程的全部成绩，管理员可见全平台成绩
- 响应数据：单科成绩、总成绩、平均分、排名、分数段直方图、学情统计数据
6. **API-006 通知查询接口**
- 请求方式：GET/POST
- 接口功能：查询用户站内通知、公告，标记通知已读/未读，清空已读通知
- 权限要求：已登录用户，仅可查看自己的通知与权限范围内的公告
- 业务规则：重要通知默认标记未读，首页红点提示，已读状态同步更新，通知保留1年
7. **API-007 用户与角色查询接口**
- 请求方式：GET
- 接口功能：查询当前登录用户信息、角色权限、菜单列表，校验页面访问权限
- 权限要求：已登录用户，管理员可查询全平台用户基础信息
- 业务规则：禁止返回用户敏感信息，普通用户仅可查看自身信息
8. **API-008 评测结果查询接口**
- 请求方式：GET
- 接口功能：查询编程实训作业的自动评测结果、运行状态、错误信息、测试用例通过情况
- 权限要求：提交作业的学生、对应课程授课教师可访问
- 业务规则：支持评测状态轮询，返回AC/WA/CE/RE/TLE等标准判题结果，评测失败给出明确提示
9. **API-009 AI 辅助问答接口**
- 请求方式：POST
- 接口功能：接收学生提问、课程上下文，返回AI答疑、知识点解释、笔记总结、思维导图生成结果
- 权限要求：已登录学生、教师可访问
- 业务规则：保留会话记录，支持上下文关联，第三方服务不可用时返回降级提示
#### 3.5.3 接口通用基础规范
1. **通信协议**：所有外部接口统一采用HTTPS协议传输，生产环境强制禁用HTTP，保障传输安全；接口请求与响应统一采用JSON格式，文件上传采用form-data格式。
2. **请求方法规范**：查询类操作统一使用GET请求，新增/修改/删除类操作统一使用POST请求，禁止使用GET请求执行数据修改操作。
3. **统一响应格式**：所有接口返回固定结构，包含状态码code、提示信息msg、业务数据data三部分。
- 成功状态码固定为200，业务异常状态码为4xx，系统异常状态码为5xx，全局统一错误码字典，禁止各模块自定义错误码。
- 分页接口统一规范请求参数pageNum、pageSize，返回参数total、pages、list，格式全局统一。
4. **权限校验规范**：所有需登录访问的接口，必须在请求头携带token身份凭证，系统自动校验登录状态、角色权限与数据访问范围，无权限、过期token直接返回401/403错误，禁止越权访问。
5. **超时与重试规范**：所有接口默认超时时间10s，第三方服务对接接口超时时间30s，支持最多2次自动重试，重试失败返回明确降级提示，不阻塞前端页面操作。
### 3.6 CSCI内部接口需求

- 用户模块与课程模块之间需要进行身份与权限校验。
- 课程模块与作业模块之间需要共享课程标识和班级标识。
- 作业模块与成绩模块之间需要传递提交结果与评分结果。
- 通知模块需要监听登录、发布、提交、批改等关键事件。
- 评测模块需要接收作业提交内容、题目配置、测试用例和运行限制。
- AI 模块需要接收用户输入内容、上下文信息和返回格式要求。
- 所有模块之间应避免直接访问对方数据库表，尽量通过服务接口或统一 Service 层交互。

<a id="sec-3-7"></a>
### 3.7 CSCI内部数据需求

1. **用户数据**
    - 用户账号信息：用户名、密码、姓名、身份、头像等。
    - 学生信息：学号、班级、专业等。
    - 教师信息：工号、所属学院、职称等。

2. **课程数据**
    - 课程编号、课程名称、课程简介、课程封面、创建者、所属班级。

3. **作业数据**
    - 作业编号、作业标题、内容、截止时间、发布教师、所属课程。

4. **提交数据**
    - 提交编号、学生、作业、提交内容、提交时间、批改结果、得分。

5. **考试数据**
    - 考试编号、考试标题、时间安排、题目内容、成绩信息。

6. **通知数据**
    - 通知标题、通知内容、通知时间、关联对象、阅读状态。

7. **系统配置数据**
    - 菜单配置、角色配置、首页布局配置、日志信息。

8. **评测数据**
    - 提交任务编号、题目编号、提交代码、运行语言、评测状态、结果详情、失败原因。

9. **AI 交互数据**
    - 会话编号、提问内容、回答内容、上下文摘要、调用时间、调用结果。

数据管理要求：

- 核心数据应以数据库持久化保存，重要操作应记录日志。
- 高频查询数据可考虑缓存或分页。
- 状态类数据应有明确枚举值，便于页面展示和业务判断。

<a id="sec-3-8"></a>
### 3.8 适应性需求

系统应能够根据不同身份展示不同首页内容，并能够适应后续扩展功能，如实验管理、题库管理、AI 辅助学习、代码评测服务接入等模块。

<a id="sec-3-9"></a>
### 3.9 保密性需求

- 用户密码应加密存储。
- 用户信息应限制在授权范围内访问。
- 系统日志中不应记录敏感明文信息。

<a id="sec-3-10"></a>
### 3.10 保密性和私密性需求

- 系统应对师生个人信息进行最小范围展示。
- 教师仅可访问自己创建的课程数据。
- 学生仅可访问自己选课、作业、成绩相关数据。
- 后续如引入更多隐私信息，需进一步完善权限控制与脱敏展示。

<a id="sec-3-11"></a>
### 3.11 CSCI环境需求

**硬件需求**：

- 服务器：用于部署核心业务服务、页面访问与接口处理。
- 数据库服务器：用于存储用户、课程、作业、成绩等数据。
- 网络设备：用于保障访问稳定性，若接入外部评测或 AI 服务，应具备稳定出网能力。
- 开发终端：用于多人协作开发与联调测试。

**操作系统**：

- 服务器端可部署于 Windows 或 Linux 环境。
- 开发与测试环境建议保持一致，便于问题定位。
- 若后续采用云服务器，应保证系统具备基本远程部署能力。

<a id="sec-3-12"></a>
### 3.12 计算机资源需求

#### 3.12.1 计算机硬件需求

- 至少具备可运行 Spring Boot 与 MySQL 的开发环境。
- 服务器应保证基本 CPU、内存和磁盘空间充足。
- 若接入外部 API，网络带宽应满足常规页面访问和接口请求需求。

#### 3.12.2 计算机硬件资源利用需求

- 合理控制页面资源和后端接口响应时间。
- 课程列表、作业列表等高频查询数据可考虑缓存或分页展示。
- 对于耗时操作应采用异步或分步处理，避免阻塞主页面操作。

#### 3.12.4 计算机通信需求

- 前后端通过 HTTP/HTTPS 通信。
- 页面操作与接口调用应保证基本的网络异常提示。
- 对外部评测服务或 AI 服务的通信需支持超时设置、失败重试和错误提示。

<a id="sec-3-13"></a>
### 3.13 软件质量因素

- **功能性**：系统应实现课程、作业、考试、成绩和通知的核心功能。
- **可靠性**：系统在普通教学负载下应保持稳定运行。
- **可维护性**：代码结构清晰、模块划分合理、命名规范。
- **可用性**：界面简洁，登录后可快速进入对应角色首页。
- **灵活性**：后续可扩展更多教学功能。
- **可移植性**：项目可在不同开发环境中部署运行。
- **可重用性**：公共组件、请求封装、菜单组件可复用。
- **可测试性**：各模块应易于单元测试和页面测试。
- **易用性**：首页导航清晰明确，操作路径尽量短。
- **安全性**：系统应支持基础认证、授权和敏感数据保护。
- **扩展性**：后续可平滑增加评测、AI、更多教学辅助功能。

<a id="sec-3-14"></a>
### 3.14 设计和实现的约束

- 后端采用 Java 8 和 Spring Boot。

### 3.15 数据

系统需要处理的数据类型包括：用户信息、课程信息、作业信息、考试信息、提交记录、成绩信息、通知信息、系统配置、操作日志以及外部服务交互数据。
数据应保证：
- **结构清晰**：所有数据模型应严格遵循统一的数据库设计规范。
- **字段含义明确**：关键字段（如状态、类型）应使用枚举值，并在数据字典中统一说明。
- **便于扩展**：数据模型设计应考虑未来功能扩展，如新增实验类型、评测方式等。
- **支持基础统计分析**：数据应能支持成绩、选课、学习进度等基础统计。
- **权限边界清晰**：数据访问必须严格遵循角色权限控制，学生、教师、管理员的数据范围界限清晰。
- **重要业务数据支持备份与恢复**：用户信息、课程信息、成绩等核心数据需定期备份。
- **支撑外部服务调用**：- **评测数据**：需包含 `评测任务ID`、`被评代码`、`运行语言`、`测试用例列表（输入/预期输出）`、`资源限制（时间、内存）`、`评测状态（PENDING、AC、WA、CE、RE、TLE）`、`第三方API返回的原始结果`。
	- **AI 交互数据**：需包含 `会话ID`、`提问内容`、`回答内容`、`上下文摘要`、`调用的AI模型`、`调用状态`、`调用耗时`。
	- **外部 API 配置数据**：需包含 `API Key`（加密存储）、`API Endpoint URL`、`超时时间`、`重试策略`、`备用API Key`等。

### 3.16 操作

系统主要操作包括：
- **通用操作**：访问首页、登录与身份判断、通知查看、用户管理、系统配置。
- **课程管理操作**：教师创建、修改、查询、归档课程；学生选课、退课。
- **作业与考试管理操作**：
	- **教师操作**：布置（包含配置测试用例）、修改、删除作业/考试；查看学生提交列表并进行在线批改（主观题）、手动复审并修改分数。
	- **学生操作**：查看作业/考试列表与详情
		- **客观题**：直接选择/填写答案并提交，系统即时返回或不返回批改结果。
		- **编程题**：在 WebIDE 中编写代码，点击提交后，系统异步调用第三方评测API，学生可查看“评测中”状态，待返回结果后查看“通过/失败”及详细信息。
		- **主观题**：上传文件或输入文本，等待教师批改。
- **成绩统计操作**：查看成绩列表、统计分布、导出成绩单。
- **外部服务交互操作**：
	- **发起评测**：系统将代码、测试用例、资源限制等封装为请求，调用第三方代码评测API。
	- **查询评测结果**：系统轮询或处理第三方API的回调，获取评测状态和结果。
	- **发起AI请求**：学生向AI助教提问，系统将问题与上下文发送给大模型API，并返回回答。
- **错误处理操作**：当第三方API调用失败或超时时，系统自动执行重试策略，并向用户显示“服务繁忙，请稍后重试”等友好提示。

<a id="sec-3-17"></a>
### 3.17 故障处理

当出现以下情况时，系统应给出明确提示并进行相应处理：

- 登录失败
- 页面无权限访问
- 数据为空或无课程可展示
- 接口请求失败
- 网络异常
- 后端服务异常
- 外部评测/AI 服务返回错误

**故障处理原则：**

1. **优先保证页面可用**：任何后端接口或外部服务故障，不应导致前端页面崩溃或白屏。
2. **给出易理解的错误提示**：避免直接暴露技术类错误信息给用户。
3. **保留日志**：记录详细的故障原因，便于开发人员排查。
4. **对外部服务调用失败时应允许重试**：系统应内置重试机制，并对用户透明

<a id="sec-3-18"></a>
### 3.18 算法说明

- **客观题自动判分算法**
	- **适用题型**：单选题、多选题、判断题、填空题（精确匹配模式）。
	- **核心逻辑**：
	1. **单选题/判断题**：学生答案与标准答案进行精确字符串匹配。若一致，得满分；否则不得分。
	2. **多选题**：将学生答案集合与标准答案集合进行对比：
		- 完全一致：得满分；
		- 学生答案属于标准答案子集（少选）：按教师配置规则（如每少一个选项扣一定分数）扣分；
		- 学生答案包含错误选项：本题不得分。
	3. **填空题（精确匹配）**：去除首尾空格后，进行大小写不敏感的字符串比较（可配置），匹配则得分。

- **编程题评测结果映射与得分算法**
	- **前提**：评测结果由第三方 API 返回，包括整体状态和每个测试用例的通过情况。
	- **得分规则**：
		- 系统将每个测试用例的权重配置为 `weight_i`，总权重和为 `totalWeight`。
		- 若某个测试用例通过（状态为 `AC`），则该用例得分 `score_i = fullScore * (weight_i / totalWeight)`。
		- 最终得分 `finalScore = sum(score_i for all passed cases)`。
	- **结果状态映射**：
		- 第三方返回 `AC` → 系统状态：`ACCEPTED`，得分计入。
		- 第三方返回 `WA` → 系统状态：`WRONG_ANSWER`，得分不计。
		- 第三方返回 `CE` → 系统状态：`COMPILE_ERROR`，得分为0，记录编译错误信息。
		- 第三方返回 `TLE` / `RE` / `MLE` → 系统状态相应映射，得分为0，记录错误类型。
	- **重试逻辑**：若第三方 API 返回网络超时或服务端错误（如5xx），系统进行最多2次重试，重试间隔2秒。重试仍失败，则标记评测状态为 `EVALUATION_FAILED`，得分为0，提示用户“评测服务异常”。

- **成绩加权统计算法**
	- **输入**：各次作业/考试得分 `score_i` 及对应的权重 `weight_i`（教师设定）。
	- **算法**：加权平均分 `finalScore = Σ(score_i * weight_i) / Σ(weight_i)`。
	- **分段统计**：
		- 将全班成绩按分数段（如 [0,60)、[60,70)、[70,80)、[80,90)、[90,100]）统计人数和占比。
		- 支持计算全班平均分、最高分、最低分、及格率。
